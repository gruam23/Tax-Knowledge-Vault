# AI 操作手册

> **任何 AI 在操作本知识库之前，必须先读完本文件。**
> 本文件是你的"飞行检查单"，每次操作完毕前必须逐项核对。

## 读取顺序（每次对话开始时）

1. 读本文件（你正在读）
2. 读 `wiki/hot.md`（了解近期上下文）
3. 读 `purpose.md`（了解研究方向）
4. 如需定位页面，读 `index.md`

## 操作后检查单（Ingest / 消化素材）

每次创建或修改 wiki 页面后，**必须在结束前完成以下全部步骤**：

- [ ] **同步更新相关 md 文件** — 每次往 wiki 中存入新知识时，必须同步更新 `wiki/` 目录下相关的 `.md` 文件（如在既有页面中补充新知识的维基链接），避免知识断层
- [ ] `wiki/hot.md` — **覆写**，更新"最后更新"和"近期变更"
- [ ] `index.md` — 新页面是否已添加到对应 section？统计数字是否正确？
- [ ] `log.md` — **每次操作结束前必须追加一条新记录**（日期 + 操作类型 + 创建/更新了哪些页面），即使今天已有其他记录也不能省略
- [ ] `.wiki-cache.json` — 如果消化了新素材，将素材签名追加到 sources 数组
- [ ] **运行健康检查** — 执行 `python scripts/lint_wiki.py`，确保没有产生孤立页面或缺少 frontmatter

## 页面规范速查

- **Frontmatter 必填**：type, title, **summary**, created, updated, status, tags, related, sources
- **summary 字段**：一句话概括页面核心内容（≤20 tokens），用于 AI 扫描全库时快速定位
- **status 取值**：seed → developing → mature → evergreen
- **文件命名**：中文短标题，不加日期前缀（素材摘要除外）
- **交叉引用**：用 `[[中文页名]]` wikilink 格式
- **来源标注**：每个知识点标注来源页面

### 三层渐进式详略架构

每个 wiki 页面必须包含三层信息，由浅入深：

| 层级      | 位置                       | Token 预算    | AI 何时读     |
| ------- | ------------------------ | ----------- | ---------- |
| Layer 1 | frontmatter `summary` 字段 | ≤20 tokens  | 扫描全库定位时    |
| Layer 2 | 正文 `## 速览` section       | ≤100 tokens | 需要初步了解时    |
| Layer 3 | 正文其余部分                   | 不限          | 深度研究/面试准备时 |

### 深度阅读指针

每个概念页的“参考素材”或“参见”章节中，必须标注原材料的**精确位置**（文件名 + 页码/章节），并使用 `[[raw文件名|显示文本]]` 格式添加**可点击的 wikilink**，指向 `raw/` 目录下的原文文件：

```markdown
## 参考素材
- [[2026-04-28-OECD-TPG-2022]] — Chapter II (pp.93-147)
  - CUP 方法详解 (pp.97-101): → [[2022-OECD-TPG-extracted-61-150|原文 pp.61-150]]
  - 利润分割法 (pp.128-147): → [[2022-OECD-TPG-extracted-61-150|原文 pp.61-150]]
```

这样用户在 Obsidian 中可以直接点击跳转到原文，AI 也能按页码回溯 `raw/` 目录下的原文，避免信息丢失。

## 目录结构速查

```
wiki/
  sources/    → 素材摘要（1篇素材1个文件）
  entities/   → 实体词条（人物/机构/法规）
  concepts/   → 概念词条（理论/模型/框架）
  topics/     → 宏观主题（跨素材聚合，素材≥2时再创建）
  comparisons/ → 对比分析
  synthesis/  → 深度报告
  queries/    → 保存的查询结果
  meta/       → 仪表盘 + lint 报告
  hot.md      → 热缓存（最重要，每次必更新）
```
